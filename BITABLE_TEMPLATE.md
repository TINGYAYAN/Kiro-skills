# 飞书多维表格模板

批量生成 APL 函数时，推荐直接按下面模板建飞书多维表格。

如果已经有一张固定模板表，优先把它配置到 `config.local.yml -> feishu.template_table_url`。之后机器人产出的批量模板说明会直接引用这张固定表，而不是临时新建数据表。

如果你想直接导入飞书，而不是手动建列，可以直接使用这两个文件：

- [bitable_template_blank.csv](/Users/yanye/code/test拨号/_tools/bitable_template_blank.csv) : 空白模板，只有表头
- [bitable_template_example.csv](/Users/yanye/code/test拨号/_tools/bitable_template_example.csv) : 带 3 行示例数据的模板

## 推荐列

| 列名 | 是否必填 | 说明 |
|---|---|---|
| 描述 | 必填 | 函数需求正文，支持多行 |
| 绑定对象 | 推荐 | 对象中文名或 API 名，如 `客户`、`AccountObj` |
| 函数类型 | 推荐 | 如 `流程函数`、`范围规则`、`计划任务`、`自定义控制器` |
| 项目 | 推荐 | 多项目场景建议明确填写，如 `朗润生物`、`西门子` |
| 函数名 | 自动 | 保持留空；生成成功后自动回填 |
| 系统API名 | 自动 | 保持留空；部署成功后自动回填 |
| 状态 | 自动 | `⏳待执行 / 🔄执行中 / ✅成功 / ❌失败` |
| 执行时间 | 自动 | 最近一次执行时间 |
| 执行反馈 | 自动 | 成功摘要或失败原因 |
| 风险级别 | 自动 | `低 / 中 / 高` |
| 人工处理建议 | 自动 | 需要人工复核或修正时给出建议 |

推荐列顺序：

```text
描述 | 绑定对象 | 函数类型 | 项目 | 函数名 | 系统API名 | 状态 | 执行时间 | 执行反馈 | 风险级别 | 人工处理建议
```

## 待执行识别规则

系统默认把下面这样的行识别为“待执行”：

- `描述` 非空
- `函数名` 为空
- `系统API名` 为空或未填

建议：

- 想重新生成一条需求时，新建一行最稳
- 如果要重跑原行，清空 `函数名` 和 `系统API名`
