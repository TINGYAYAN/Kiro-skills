# APL 自动化流水线 —— 能力总览

> 写清楚需求，就能自动生成代码、自动部署到纷享、自动跑测试。核心是：**像人一样操作浏览器**，把「生成 → 部署 → 验证」一口气跑完。

---

## 一、能做什么

| 能力 | 一句话说明 |
|------|------------|
| **需求变代码** | 用自然语言描述业务，AI 生成符合纷享规范的函数代码 |
| **代码自动上线** | 自动登录纷享、新建函数、填表、粘贴代码、保存，全程无需人工点鼠标 |
| **自动查字段** | 自动打开纷享编辑器里的字段对照表，把对象字段信息抓下来供生成用 |
| **自动测试** | 自动造数据、触发函数、检查结果、清理数据 |
| **飞书联动** | 部署成功后自动记到飞书表格；批量模式从飞书写需求、执行完写回结果 |
| **批量处理** | 在飞书表格里填一堆需求，一条消息触发，逐个生成并部署 |

---

## 二、整体流程

```
写需求（req.yml）
      ↓
自动抓字段（可选，首次或刷新时用）
      ↓
AI 生成代码
      ↓
自动部署到纷享（★ 核心：模拟人操作浏览器）
      ↓
自动测试（可选）
      ↓
记到飞书（可选）
```

---

## 三、怎么用

**单条需求：**

1. 写一个 `req.yml`，把需求、对象、函数类型等填好
2. 运行一条命令，等它跑完
3. 纷享里就有新函数了

**批量需求：**

1. 在飞书多维表格里填「描述」「绑定对象」，函数名留空
2. 给机器人发「批量生成」
3. 机器人逐个处理，把函数名、API 名、成功/失败写回表格

---

## 四、出错了怎么办

- **找不到按钮/输入框**：会自动截图保存，告诉你截图在哪，方便对照页面排查
- **下拉选不到**：会打印当前有哪些选项，方便你改配置
- **部署失败**：会说明是哪一步失败、为什么，而不是只报一个笼统错误

遇到已知问题，可以看 `TROUBLESHOOTING.md`，里面有常见坑和解决办法。

---

## 五、数据怎么流转

**批量模式：**

飞书表格里写需求 → 工具读取 → 生成 + 部署 → 把结果写回同一行（函数名、API 名、成功/失败）

这样需求从哪来、结果到哪去，都在飞书里，一目了然。

## 六、能省多少事

| 场景 | 以前 | 现在 |
|------|------|------|
| 新建一个流程函数 | 手写代码 → 登录纷享 → 新建 → 填表 → 粘贴 → 保存 | 写需求文件 → 一条命令 |
| 一堆需求要处理 | 一个一个复制到对话里、一个一个部署 | 飞书填表 → 批量生成 |
| 查对象有哪些字段 | 翻文档、问实施 | 自动抓取，直接可用 |
| 验证函数是否正常 | 手动点界面测 | 写好用例，自动跑 |

---

## 七、和普通「对话生成代码」有啥不一样

| 对比 | 普通对话（如 Cursor） | 本工具 |
|------|------------------------|--------|
| 生成完代码之后 | 你得自己登录纷享、填表、粘贴 | **自动帮你做完**，直接上线 |
| 字段信息 | 得自己查文档或问人 | **自动从纷享里抓**，和系统一致 |
| 验证 | 一般靠人工试 | **自动跑测试**，结果可复现 |
| 批量 | 一条一条聊、一条一条处理 | **飞书表格驱动**，一批一批跑 |
| 出错 | 多半要你描述现象 | **自动截图、打印选项**，方便自己排查 |

**一句话**：普通生成只给你代码；这里会一直帮你做到「纷享里能跑起来」为止。

---

## 八、为什么必须用「模拟操作浏览器」

纷享的函数管理页面**没有对外开放接口**，只能通过网页操作：

- 登录、填表、选下拉、点保存
- 在代码编辑器里粘贴
- 在编辑器里看字段对照表

所以只能「像人一样操作浏览器」——这就是 Playwright 做的事。它负责：

- 自动登录、记住登录状态
- 在各种表单、下拉、弹窗里正确点击和填写
- 把代码贴进编辑器并保存
- 出问题时自动截图，方便排查

没有这一步，代码生成完了还得人工去点，就谈不上「端到端自动化」。

## 九、快速上手

```bash
# 安装
pip install -r requirements.txt
playwright install chromium

# 配置（填纷享账号、AI 接口、飞书等）
cp config.yml config.local.yml

# 跑一条完整流水线
python pipeline.py --req req.yml --step all
```
